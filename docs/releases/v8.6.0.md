# NOOP v8.6.0

An accuracy pass on overnight HRV — the number, and the recovery built on it, now read true — plus a rebuilt Latest Workouts feed. Everything runs on your own device, offline, no account. Update from the Releases page, AltStore, or `brew upgrade`.

## Fixed

- **Overnight HRV no longer reads roughly twice as high as it should (#195).** HRV is built from the change between one heartbeat and the next. When the cleaning step removes a noisy beat, its two neighbours become adjacent — and the old code counted the jump across that gap as a real beat-to-beat change, which (because the measure squares each change) let one removed beat dominate the whole night and push HRV high. Cleaning is now gap-aware: a change that spans a removed beat is skipped, so a dropped beat can't manufacture a spike. On clean nights nothing changes; noisy nights that read ~2× high come back down to a true value, and recovery follows. iPhone, Mac and Android, computed identically on each.
- **Switching the HRV window no longer resets Charge to "calibrating" (#201).** Moving between whole-night and deep-sleep HRV used to throw away your baseline history and restart the several-night calibration, so an established user briefly lost their Charge score and it read as though the setting was broken. The switch now re-scores your recent nights over the new window and keeps a usable baseline immediately, as long as you have a few nights of history. A genuine first-week user still calibrates honestly. Thanks digitalerdude.
- **Editing or re-syncing a past night respects your deep-sleep HRV setting (Android) (#195).** A night that was edited or back-filled was re-scored over the whole night even when you'd chosen the deep-sleep window; it now honours the window you picked, matching iPhone and Mac. Thanks digitalerdude.
- **Latest Workouts shows the true most-recent sessions (#200).** The Today workout section was pinned to the original strap id, so after re-pairing it stranded your newest recordings; it could also show the same session twice when it arrived from more than one source (Health Connect, a manual entry, and the strap), and some sports — volleyball and other imported types — showed as a generic "Workout." It now follows your active strap, removes the cross-source duplicates, names the full set of sports, and refreshes as soon as an import lands. Android catch-up to what iPhone and Mac already did. Thanks TheBoroer.

## Changed

- **"Last Workouts" is now "Latest Workouts," as a single clean list.** The section header read as "final" rather than "most recent," and the old 2×2 grid interleaved sources; it's now a single-column list of your four newest workouts with a small source tag on imported ones. Both platforms ("Neueste Workouts" in German).

## Notes

- **Every scored night now writes a one-line HRV cleaning summary to the strap log (#195).** A "my HRV looks wrong" report can now be triaged from a Test Centre export without turning on a special mode — the line records the cleaned HRV, how many heartbeats survived cleaning, and how many were rejected. It's diagnostic only and never changes your scores.
